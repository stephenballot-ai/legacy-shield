## De Code Die Niemand Weet Dat Je Hebt

Je codeert al 15 jaar. Je GitHub-account bevat:

- **Privérepositories** met jarenlange side projects en frameworks die je zelf hebt gebouwd
- **Intellectueel eigendom** dat je wilde monetariseren of open-sourcen
- **Bedrijfsgeheimen** voor klanten of je eigen startup
- **Persoonlijke scripts** die je leven automatiseren
- **Documentatie** die nergens anders bestaat
- **Configuratiebestanden** met infrastructuurkennis die alleen jij bezit

Als je morgen sterft, wordt het allemaal ontoegankelijk. Niet verloren — maar vergrendeld achter je gebruikersnaam en wachtwoord, waar niemand meer bij kan.

Je familie weet niet eens dat deze repositories bestaan. Je zakenpartner kan de code niet benaderen. De open-source-gemeenschap verliest het gereedschap dat je aan het bouwen was. Je levenswerk verdwijnt digitaal.

## Waarom GitHub Je Familie Niet Kan Helpen

GitHub heeft geen "legacy access"-functie zoals Google of Facebook. Er is geen manier om een opvolger aan te wijzen. Er is geen proces voor het verifiëren van overlijden en het overdragen van accounteigenaarschap.

Dit is wat je familie tegemoet gaat:

**Ze hebben je wachtwoord nodig.** Zelfs als ze het op de een of andere manier krijgen, blokkeren twee-factor-authenticatie (die GitHub vereist) alles. Je authenticatie-app staat op je telefoon. Je backup-codes zitten in je wachtwoordmanager. Beide zijn ontoegankelijk.

**GitHub Support kan niet helpen.** GitHub Support heeft gesteld dat ze geen accounteigenaarschap kunnen overdragen, zelfs niet met een overlijdenscertificaat. Ze stellen ook geen legacy access in. Jij bent niet hun prioriteit — actieve betalende gebruikers zijn dat wel.

**De repositories verdwijnen niet.** GitHub gaat nergens heen. Je privécode zit daar gewoon, volledig privé, vrijwel waardeloos omdat niemand levend weet dat het bestaat of hoe je het kunt benaderen.

## Waarom Dit Belangrijker Is Dan Je Denkt

### Je Hebt Open-Source Verantwoordingen

Misschien onderhoud je een populair npm-pakket of een Python-bibliotheek. Honderden of duizenden developers zijn ervan afhankelijk. Als je sterft en niemand kan de repository benaderen, verliest de gemeenschap de maintainer.

Beveiligingskwetsbaarheden blijven ongepatched. Bug reports stapelen zich op zonder reacties. Fork-problemen vermenigvuldigen zich terwijl developers naar alternatieven zoeken. Je bijdrage aan open-source wordt een waarschuwingsverhaal.

### Je Hebt Verborgen Bedrijfswaarde

Je code kan geld waard zijn. Een GitHub-repository met een nuttig hulpmiddel, framework of SaaS-codebase kan waardevol intellectueel eigendom zijn. Als je het wilde verkopen, licenseren of als open-source uitbrengen, voorkomt je dood dat alles daarvan gebeurt.

Je familie erft niets. De code sterft met je mee.

### Je Hebt Infrastructuurkennis Die Alleen Jij Hebt

Je deployment-scripts, configuratiebestanden, monitoring-setup en infrastructure-as-code repositories bevatten kritieke kennis. Als je GitHub het enige exemplaar van je Terraform-configuraties, Docker-setup of CI/CD-pipelines bevat, creëer je een kennissilo die sterft met je.

## Het Digitale Erfenis Problem Voor Developers

**Denk aan het verhaal van Marcus**, een 42-jarige developer in Amsterdam. Hij bouwde een succesvolle SaaS-onderneming met code volledig op GitHub — alleen privérepositories. Zijn zakenpartner had toegang tot het AWS-account maar niet tot de GitHub-repos. Toen Marcus een plotselinge beroerte kreeg en drie maanden werd opgeheven, zat zijn zakenpartner vast. Kritieke bugfixes konden niet worden uitgerold. Het SaaS-bedrijf stagneerde. Klanten vertrokken.

Toen Marcus herstelde, was hij woedend over de kwetsbaarheid. Maar de meeste developers herstellen niet. Ze sterven.

**Of denk aan Sofia**, een freelance developer in Rotterdam. Ze onderhield verschillende open-source projecten die industry-standaardtools waren geworden. Haar GitHub was haar portfolio, haar inkomen, haar reputatie. Toen zij onverwacht op 38-jarige leeftijd overleed, hadden duizenden developers plotseling verweesde afhankelijkheden. Enterprise-projecten die op haar code vertrouwden, hadden beveiligingskwetsbaarheden zonder reparatiepad.

Dit zijn geen uitzonderingen. Dit zijn waarschuwingen.

## Wat Je Vandaag Moet Doen

### 1. Maak een Gedocumenteerde Code-Inventaris

Maak een lijst van elke repository die belangrijk is:

- Privéprojecten met bedrijfswaarde of persoonlijk belang
- Open-source projecten die je onderhoudt
- Bedrijfscode voor klanten of je bedrijf
- Infrastructuurrepositories
- Hulpprogramma's die je hebt gemaakt

Noteer voor elk project:
- De repository-URL
- Het doel en huidige status
- Eventuele bedrijfswaarde of licentie-informatie
- Wie na je dood toegang moet hebben
- Wat ermee moet gebeuren (archiveren, overdragen aan organisatie, open-sourcen, verwijderen)

**Zet deze lijst in je digitale testament.** Niet in je GitHub-notities — maar in LegacyShield of met je juridische documenten.

### 2. Draag Eigenaarschap Over aan een Organisatie

De beste praktijk voor code-erfenis is het aanmaken van een GitHub-organisatie en jezelf en een vertrouwde persoon beide als eigenaar instellen. Als je sterft, behoudt je opvolger toegang.

Stappen:
1. Maak een GitHub-organisatie voor je belangrijke repositories
2. Voeg je zakenpartner, mede-eigenaar of vertrouwde collega als eigenaar toe
3. Verplaats je belangrijke privérepositories naar de organisatie
4. Documenteer dit in je digitale testament

Organisaties overleven individuen. Als beide eigenaren sterven, zal GitHub het uiteindelijk sluiten, maar er is tenminste een weg vooruit.

### 3. Documenteer Je GitHub-Toegang

Je digitale executeur heeft nodig:
- Je GitHub-gebruikersnaam
- Instructies waar je wachtwoord te vinden is (in je wachtwoordmanager's "digitale executeur" toegangssectie)
- Recovery-codes voor twee-factor-authenticatie veilig opgeslagen
- Een notitie die het doel van elke belangrijke repository uitlegt

### 4. Plan Voor Open-Source Projecten

Als je open-source-projecten onderhoudt:
- Documenteer de projecten en hun belang
- Wijs een opvolgingsonderhouder aan in de README
- Overweeg over te dragen aan een organisatie of erkende stichting
- Plan voor geautomatiseerde beveiligingsupdates na je dood

### 5. Sla Je GitHub Recovery-Codes Op

Je GitHub-recovery-codes zijn je laatste redmiddel als je je telefoon of app kwijtraakt. Sla ze op in:
- Je LegacyShield-beveiligde kluis
- Je wachtwoordmanager's noodomstandigheidstoegang
- Geprint en opgeslagen met je testament
- **Gedeeld met je digitale executeur** via een veilige methode

Als niemand je recovery-codes heeft, is je GitHub-account werkelijk vergrendeld voor altijd.
