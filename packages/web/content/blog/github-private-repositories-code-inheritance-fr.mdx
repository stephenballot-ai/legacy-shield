## Le Code Dont Personne Ne Connaît l'Existence

Vous codez depuis 15 ans. Votre compte GitHub contient:

- **Des repositories privées** avec des années de projets secondaires et de frameworks que vous avez construits
- **De la propriété intellectuelle** que vous aviez l'intention de monétiser ou de rendre open-source
- **Du code propriétaire** pour des clients ou votre propre startup
- **Des scripts personnels** qui automatisent votre vie
- **De la documentation** qui n'existe nulle part ailleurs
- **Des fichiers de configuration** avec des connaissances en infrastructure que seul vous possédez

Si vous mourez demain, tout cela devient inaccessible. Pas perdu — mais verrouillé derrière votre nom d'utilisateur et votre mot de passe, accumalant la poussière numérique à jamais.

Votre famille ne sait même pas que ces repositories existent. Votre partenaire commercial ne peut pas accéder au code. La communauté open-source perd l'outil que vous construisiez. Votre travail de vie disparaît numériquement.

## Pourquoi GitHub Ne Peut Pas Aider Votre Famille

GitHub n'a pas de fonction "accès patrimonial" comme Google ou Facebook l'offrent. Il n'y a pas de moyen de désigner un successeur. Il n'existe pas de processus pour vérifier le décès et transférer la propriété du compte.

Voici ce que votre famille doit affronter:

**Elle a besoin de votre mot de passe.** Même s'ils le obtiennent d'une manière ou d'une autre, l'authentification à deux facteurs (que GitHub exige) bloque tout. Votre application d'authentification est sur votre téléphone. Vos codes de récupération sont dans votre gestionnaire de mots de passe. Les deux sont inaccessibles.

**Contacter le support GitHub est futile.** Le support GitHub a déclaré qu'il ne pouvait pas transférer la propriété du compte, même avec un certificat de décès. Ils ne mettent pas non plus en place l'accès hérité. Vous n'êtes pas leur priorité — les utilisateurs payants actifs le sont.

**Les repositories ne disparaissent pas.** GitHub ne va nulle part. Votre code privé reste là, complètement privé, essentiellement inutile parce que personne de vivant ne sait qu'il existe ou comment y accéder.

## Pourquoi C'est Plus Important Que Vous Ne Le Pensez

### Vous Avez Des Obligations Open-Source

Peut-être maintenez-vous un package npm populaire ou une bibliothèque Python. Des centaines ou des milliers de développeurs en dépendent. Quand vous mourez et que personne ne peut accéder au repository, la communauté perd le responsable.

Les vulnérabilités de sécurité restent sans correction. Les rapports de bugs s'accumulent sans réponses. Les problèmes de bifurcation se multiplient pendant que les développeurs cherchent des alternatives. Votre contribution à l'open-source devient un conte d'avertissement.

### Vous Avez Une Valeur Commerciale Cachée

Votre code pourrait valoir de l'argent. Un repository GitHub avec un utilitaire utile, un framework ou une base de code SaaS pourrait être une propriété intellectuelle précieuse. Si vous aviez l'intention de le vendre, de le concéder sous licence ou de le publier en open-source, votre mort l'empêche.

Votre famille n'hérite de rien. Le code meurt avec vous.

### Vous Possédez Des Connaissances en Infrastructure Que Seul Vous Possédez

Vos scripts de déploiement, fichiers de configuration, configurations de surveillance et repositories d'infrastructure-en-tant-que-code contiennent des connaissances critiques. Si votre GitHub contient le seul exemplaire de vos configurations Terraform, votre configuration Docker ou vos pipelines CI/CD, vous créez un silo de connaissances qui meurt avec vous.

## Le Problème de l'Héritage Numérique Pour les Développeurs

**Considérez l'histoire de Laurent**, un développeur de 42 ans à Paris. Il a construit une entreprise SaaS prospère avec du code entièrement sur GitHub — repositories privées uniquement. Son partenaire commercial avait accès au compte AWS mais pas aux repos GitHub. Quand Laurent a eu un accident vasculaire cérébral soudain et a été hospitalisé pendant trois mois, son partenaire était bloqué. Les corrections de bugs critiques ne pouvaient pas être déployées. L'entreprise SaaS a stagné. Les clients ont quitté.

Quand Laurent s'est rétabli, il était furieux de cette vulnérabilité. Mais la plupart des développeurs ne se rétablissent pas. Ils meurent.

**Ou considérez Amélie**, une développeuse indépendante à Lyon. Elle maintenait plusieurs projets open-source qui étaient devenus des outils standards de l'industrie. Son GitHub était son portfolio, ses moyens de subsistance, sa réputation. Quand elle est décédée subitement à 38 ans, des milliers de développeurs avaient soudainement des dépendances abandonnées. Les projets d'entreprise qui dépendaient de son code faisaient face à des vulnérabilités sans voie de réparation.

Ce ne sont pas des cas limites. Ce sont des avertissements.

## Ce Que Vous Devriez Faire Aujourd'hui

### 1. Créez Un Inventaire De Code Documenté

Énumérez chaque repository qui compte:

- Projets privés avec valeur commerciale ou importance personnelle
- Projets open-source que vous maintenez
- Code propriétaire pour des clients ou votre entreprise
- Repositories d'infrastructure
- Utilitaires que vous avez créés

Pour chacun, notez:
- L'URL du repository
- L'objectif et l'état actuel
- Toute valeur commerciale ou information de licence
- Qui devrait avoir accès après votre décès
- Ce qui devrait lui arriver (archiver, transférer à une organisation, open-source, supprimer)

**Mettez cette liste dans votre testament numérique.** Pas dans vos notes GitHub — mais dans LegacyShield ou avec vos documents juridiques.

### 2. Transférez La Propriété À Une Organisation

La meilleure pratique pour l'héritage du code est de créer une organisation GitHub et de vous faire nommer vous-même et une personne de confiance comme propriétaires. Quand vous mourez, votre successeur garde accès.

Étapes:
1. Créez une organisation GitHub pour vos repositories importants
2. Ajoutez votre co-fondateur, partenaire commercial ou collègue de confiance comme propriétaire
3. Transférez vos importants repositories privés à l'organisation
4. Documentez cela dans votre testament numérique

Les organisations survivent aux individus. Si les deux propriétaires meurent, GitHub fermera finalement, mais au moins il y a un chemin en avant.

### 3. Documentez Votre Accès GitHub

Votre exécuteur testamentaire numérique a besoin:
- De votre nom d'utilisateur GitHub
- D'instructions sur où trouver votre mot de passe (dans la section d'accès "exécuteur testamentaire numérique" de votre gestionnaire de mots de passe)
- De codes de récupération pour l'authentification à deux facteurs stockés en toute sécurité
- D'une note expliquant l'objectif de chaque repository important

### 4. Planifiez Pour Les Projets Open-Source

Si vous maintenez des projets open-source:
- Documentez les projets et leur importance
- Désignez un responsable successeur dans le README
- Envisagez le transfert à une organisation ou une fondation établie
- Planifiez les mises à jour de sécurité automatisées après votre décès

### 5. Stockez Vos Codes De Récupération GitHub

Vos codes de récupération GitHub sont votre dernier recours si vous perdez votre téléphone ou votre application d'authentification. Stockez-les dans:
- Votre coffre-fort sécurisé LegacyShield
- La section d'accès d'urgence de votre gestionnaire de mots de passe
- Imprimé et conservé avec votre testament
- **Partagé avec votre exécuteur testamentaire numérique** via une méthode sécurisée

Si personne n'a vos codes de récupération, votre compte GitHub est vraiment verrouillé pour toujours.

## La Réalité Française Pour Les Développeurs

En tant que développeur, votre code est souvent votre propriété intellectuelle la plus précieuse. Selon la loi française de succession, votre code tombe sous vos actifs numériques — mais il n'existe pas de mécanisme pour le transférer à moins que vous n'ayez planifié à l'avance.

En France, le *notaire* est chargé de la succession, mais il présume que vous avez documenté vos actifs numériques. La plupart des développeurs ne l'ont pas fait.

Pour les travailleurs indépendants et les entrepreneurs partout en France, votre GitHub peut représenter des années de travail valant des dizaines de milliers d'euros. Pourtant, la plupart n'ont rien fait pour assurer qu'il passe à leur famille.
